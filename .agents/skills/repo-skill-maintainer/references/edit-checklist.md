# Edit Checklist

Use this checklist when modifying the `my-skills` repository.

## Checklist

1. Inspect the target folder before writing
2. Decide whether the change belongs in:
   - `SKILL.md`
   - `references/`
   - agent-facing example files
   - `README.md`
3. Keep `SKILL.md` lean and push depth into references
4. Update agent-native examples if real-world usage changed
5. Avoid unrelated edits
6. Do a manual sanity check if automated validation is unavailable

## Good outcomes

- a clearer trigger description
- better structured references
- aligned example surfaces
- improved discoverability
